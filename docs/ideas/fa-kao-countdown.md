# 法考倒计时 - 产品定义

## 问题陈述
如何为2026年法考生（特别是边工作边备考的在职考生）创建一个极简、无干扰的倒计时应用，通过每日中国古典诗词提供情感支持，成为他们日常备考路上的精神陪伴？

## 推荐方向

**极简倒计时 + 每日诗词** —— 一个纯展示型应用，无交互，无干扰。

### 核心体验
- **第一屏**：大号倒计时显示"距离2026年法考还有XXX天"
- **第二屏**：今日诗词（标题 + 作者 + 正文）
- **第三屏**：诗词赏析/背景（可选展开）
- **无其他元素**：无按钮、无导航、无设置、无社交

### 设计理念
- **打开即用**：3秒内获得所需信息
- **被动获取**：无需用户任何输入或操作
- **情感支持**：诗词是唯一的互动，提供每日情绪价值
- **极简禅意**：减少一切可能的干扰

### 技术方案
- **推荐**：PWA（Progressive Web App）+ TWA（Trusted Web Activity）
- **优势**：
  - 可以用HTML/CSS/JS开发，无需学习React Native
  - 可打包成Android APK上架Google Play
  - 支持离线使用（Service Worker）
  - 支持推送通知（可选，未来扩展）
- **替代方案**：React Native + Expo（如果需要更好的原生体验）

## 关键假设待验证

### 用户行为假设
- **假设1**：法考生愿意每日打开一个只有倒计时和诗词的APP
  - *验证方式*：先做Web版，邀请10-20位目标用户试用1周，观察回访率

- **假设2**：每日更新的诗词足以提供情绪价值，无需社交或打卡功能
  - *验证方式*：用户访谈，询问"你觉得这个APP缺少什么吗？"

- **假设3**：在职考生更喜欢"被动查看"而非"主动记录"
  - *验证方式*：对比测试（方向A vs 方向B），观察用户偏好

### 内容假设
- **假设4**：中国古典诗词（唐宋为主）能引起法考生的情感共鸣
  - *验证方式*：A/B测试不同类型的诗词（古典 vs 现代 vs 励志金句）

- **假设5**：每日一首的更新频率是合适的
  - *验证方式*：用户反馈是否希望"更多"或"更少"

## MVP范围

### 第一版包含
- ✅ 倒计时显示（距离2026年法考还有XX天）
- ✅ 每日一首诗词（标题、作者、正文）
- ✅ 诗词点击展开显示完整版和简单赏析
- ✅ 日期自动判断（基于系统时间）
- ✅ 离线可用（PWA缓存）
- ✅ 可安装到桌面（Android APK）

### 第一版不包含
- ❌ 用户注册/登录
- ❌ 打卡功能
- ❌ 社交功能（评论、分享）
- ❌ 学习进度记录
- ❌ 诗词收藏/喜欢
- ❌ 主题切换
- ❌ 字体大小调整
- ❌ 推送通知

## 明确不做（以及为什么）

| 功能 | 为什么不做 |
|------|-----------|
| 打卡/记录 | 职考生时间碎片化，强制打卡会增加负担；且验证假设后发现"被动查看"更符合需求 |
| 社交功能 | 增加复杂度，且第一版要验证的核心是"倒计时+诗词"是否足够，而非社交 |
| 用户系统 | 无需个性化，所有用户看到相同内容；减少开发和维护成本 |
| 多主题 | 第一版专注核心价值，设计一个优雅的默认主题即可 |
| 诗词搜索/分类 | 每日只推一首，不需要选择；减少决策负担 |
| 学习计划集成 | 功能边界清晰，只做情感支持，不做学习管理 |
